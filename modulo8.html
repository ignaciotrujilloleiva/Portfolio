2. Caso de Estudio

Caso de Estudio: Alke Wallet

Aplicación web desarrollada con el Framework Django

Descripción del Proyecto

El proyecto Alke Wallet es una aplicación web desarrollada en Django para la gestión de usuarios y transacciones financieras básicas. Implementa una arquitectura MVT y una base de datos relacional para garantizar la correcta persistencia de la información. El enfoque principal fue construir un sistema funcional y bien estructurado, priorizando la lógica de negocio y la integración entre backend y frontend.

El Desafío

El principal desafío del proyecto fue implementar correctamente la integración entre el backend y la base de datos, utilizando el ORM de Django para modelar entidades relacionadas y asegurar la persistencia de la información.

Esto implicó comprender y aplicar:

  • Entendimiento de la estructura de un proyecto creado con Django.
  • Definición de modelos y relaciones.
  • Persistencia de datos mediante el ORM de Django.
  • Separación clara entre lógica, presentación y datos.
  • Renderizado dinámico de información en templates.

Solución Implementada

Se desarrolló una aplicación web desde cero utilizando Django, donde se definieron tres modelos principales: Usuario, Moneda y Transacción, conectados mediante relaciones de tipo ForeignKey, permitiendo una correcta gestión y persistencia de la información.

  • Diseño de modelos que representan entidades del sistema, permitiendo escalabilidad y mantenimiento.
  • Implementación de relaciones uno a muchos (1:N) mediante claves foráneas.
  • Uso de Django Template Language para renderizar datos dinámicos en la interfaz.
  • Integración de formularios para la creación y edición de registros.
  • Uso de choices para restringir valores en tipos de transacción, evitando errores de ingreso.
  • Implementación de related_name para facilitar consultas inversas en el ORM.
  • Definición de métodos __str__ para mejorar la legibilidad en el panel administrativo.
  • Uso del panel de administración de Django para gestión eficiente de datos.

Arquitectura del Sistema

El sistema fue desarrollado siguiendo el patrón MVT, donde:

  • Model: Define la estructura de datos y relaciones.
  • View: Maneja la lógica de negocio y flujo de datos.
  • Template: Presenta la información al usuario final.

Herramientas Utilizadas

Python, Django, SQLite, HTML, CSS, Bootstrap 5, ORM, Visual Studio Code, Github.

Aprendizajes Clave

Durante el desarrollo del proyecto se lograron consolidar conocimientos clave como:

  • Uso del ORM de Django para modelar y gestionar datos.
  • Implementación de relaciones entre entidades en bases de datos relacionales.
  • Gestión de migraciones y sincronización con la base de datos.
  • Desarrollo de operaciones CRUD completas.
  • Integración entre backend y frontend mediante templates.
  • Uso de choices para asegurar consistencia en los datos ingresados.
  • Aplicación de related_name para optimizar consultas entre modelos.
  • Mejora en la legibilidad del sistema mediante métodos __str__.

Además, se adquirió una comprensión más completa del flujo de una aplicación web, desde la definición de modelos hasta la visualización de datos en la interfaz de usuario.

Métricas de Impacto

  • Implementación de 3 modelos relacionados en base de datos
  • Desarrollo de CRUD completos para 2 entidades principales
  • Reducción de errores de ingreso mediante uso de choices
  • Optimización de consultas gracias a related_name
  • Tiempo de desarrollo optimizado mediante uso del ORM y Django Admin

Habilidades técnicas aplicadas

Justificación del Proyecto

Este proyecto fue seleccionado como caso de estudio porque representa un punto clave en mi formación, donde logré integrar por primera vez una aplicación web completa conectada a una base de datos. A diferencia de proyectos anteriores, este desarrollo implicó trabajar con persistencia de datos, relaciones entre entidades y lógica backend más compleja, lo que marcó un avance significativo en mis habilidades técnicas. Además, el proyecto refleja buenas prácticas de desarrollo y una estructura organizada, lo que lo convierte en un ejemplo representativo de mi nivel actual como desarrollador en formación.
