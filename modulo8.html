2. Caso de Estudio

Caso de Estudio: CRUD Django con Integración DataBase

Integración de Django y Gestión de Datos en el sector Sanitario

Descripción del Proyecto

Este proyecto corresponde al desarrollo de una aplicación web utilizando Django, como parte del Módulo 7 del bootcamp Fullstack Python.

El objetivo fue construir una plataforma funcional capaz de gestionar información mediante una base de datos relacional, implementando operaciones CRUD completas y siguiendo el patrón arquitectónico MVT (Model - View - Template).

El Desafío

El principal desafío fue trasladar la lógica de negocio a una aplicación web estructurada, integrando correctamente:

  • Definición de modelos y relaciones en models.py.
  • Persistencia de datos mediante el ORM de Django.
  • Separación clara entre lógica, presentación y datos.
  • Renderizado dinámico de información en templates.

Solución Implementada

Se desarrolló una arquitectura basada en buenas prácticas de Django:

  • Diseño de modelos que representan entidades del sistema, permitiendo escalabilidad y mantenimiento.
  • Implementación de vistas para manejar la lógica de negocio y conectar el backend con el frontend.
  • Uso de Django Template Language para renderizar datos dinámicos.
  • Integración de formularios y validaciones básicas.
  • Uso del panel de administración de Django para gestión eficiente de datos.

Arquitectura del Sistema

El sistema fue desarrollado siguiendo el patrón MVT, donde:

  • Model: Define la estructura de datos y relaciones.
  • View: Maneja la lógica de negocio y flujo de datos.
  • Template: Presenta la información al usuario final.

Herramientas Utilizadas

Python, Django, SQLite, HTML, CSS, Bootstrap 5, ORM.

Aprendizajes Clave
